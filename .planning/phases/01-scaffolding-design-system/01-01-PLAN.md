# Phase 1: Project Scaffolding & Design System

## Phase Overview
Set up the Next.js 14+ project with App Router, configure Supabase client, establish the design system (colors, typography, spacing), and create the global responsive layout shell.

**Budget Tier**: Standard
**Estimated Effort**: 6-8 hours
**Number of Atomic Tasks**: 3

---

<task type="auto" id="01-01">
  <n>Next.js Project Initialization & Tooling</n>
  
  <context>
    This is a greenfield Next.js project for InterviewMate AI.
    The project folder already has PRD.md and .gitignore.
    We need Next.js 14+ with App Router, TypeScript strict mode,
    ESLint, and Supabase client SDK configured.
  </context>
  
  <steps>
    1. Run `npx -y create-next-app@latest ./ --typescript --eslint --app --src-dir --import-alias "@/*" --yes --disable-git` in project root
    2. Install Supabase client: `npm install @supabase/supabase-js`
    3. Create `src/lib/supabase/client.ts` — browser client factory
    4. Create `src/lib/supabase/server.ts` — server-side client factory
    5. Create `.env.local` with placeholder Supabase keys (NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY)
    6. Add `.env.local` to .gitignore (should already be there)
    7. Configure `tsconfig.json` for strict mode
    8. Add Prettier: `npm install -D prettier eslint-config-prettier`
    9. Create `.prettierrc` with project conventions
    10. Verify: `npm run build` succeeds with zero errors
  </steps>
  
  <verify>npm run build</verify>
  
  <outcome>
    Working Next.js 14+ project with TypeScript strict mode,
    ESLint + Prettier, Supabase SDK configured, and a clean build.
  </outcome>
</task>

---

<task type="auto" id="01-02">
  <n>Design System: Tokens, Typography & Global Styles</n>
  
  <context>
    The project needs a premium, modern design system.
    Tech: Next.js with CSS (Tailwind or vanilla CSS — to be decided in this task).
    Brand: Professional, clean, dark mode support.
    Typography: Inter (primary) or Outfit (headings).
    Avatar-centric: The UI should feel alive and premium — not generic.
  </context>
  
  <steps>
    1. Decide: Use Tailwind CSS (already included via create-next-app) OR vanilla CSS.
       Recommendation: Use Tailwind CSS — it ships with create-next-app and pairs well with Next.js.
    2. Define design tokens in `tailwind.config.ts`:
       - Color palette: primary (indigo/violet gradient), neutral (slate), accent (emerald/teal), semantic (success/warning/error)
       - Typography: Inter (body), Outfit (headings) via Google Fonts
       - Spacing scale (4px base)
       - Border radius tokens
       - Shadow tokens (subtle, medium, dramatic)
       - Dark mode configuration (class-based)
    3. Create `src/styles/globals.css`:
       - CSS reset / base styles
       - Font imports (Google Fonts)
       - CSS custom properties for theme tokens
       - Dark mode defaults
       - Utility classes for glassmorphism, gradients
    4. Create `src/styles/animations.css`:
       - Micro-animation keyframes (fade-in, slide-up, scale, pulse)
       - Transition utilities
    5. Verify: `npm run build` succeeds, `npm run dev` renders styled page
  </steps>
  
  <verify>npm run build</verify>
  
  <outcome>
    Complete design system with color palette, typography, spacing tokens,
    dark mode, and animation utilities. Visually premium foundation.
  </outcome>
</task>

---

<task type="auto" id="01-03">
  <n>Global Layout Shell & Navigation</n>
  
  <context>
    Build the responsive app shell that all pages will render inside.
    Two layout modes:
    - Recruiter layout: sidebar nav + header + content area
    - Candidate/Public layout: minimal header + full-width content
    At this stage, pages can be placeholders. The goal is the structural shell.
  </context>
  
  <steps>
    1. Create `src/app/layout.tsx` — root layout with fonts, metadata, theme provider
    2. Create `src/components/layout/Header.tsx` — responsive header with logo, nav links, dark mode toggle
    3. Create `src/components/layout/Sidebar.tsx` — recruiter sidebar with navigation links (Dashboard, Interviews, Reports, Settings)
    4. Create `src/components/layout/Footer.tsx` — minimal footer
    5. Create `src/app/(recruiter)/layout.tsx` — recruiter route group with sidebar layout
    6. Create `src/app/(public)/layout.tsx` — public/candidate route group with minimal layout
    7. Create placeholder pages:
       - `src/app/(recruiter)/dashboard/page.tsx` — "Dashboard" heading
       - `src/app/(public)/page.tsx` — Landing/home page placeholder
    8. Add responsive breakpoints: mobile-first, sidebar collapses on mobile
    9. Add dark mode toggle functionality (localStorage + system preference)
    10. Verify: `npm run dev` shows responsive layout, sidebar toggles, dark mode works
    11. Commit: `feat(02-28): Phase 1 complete — scaffolding & design system`
  </steps>
  
  <verify>npm run build && npm run dev</verify>
  
  <outcome>
    Fully responsive app shell with header, sidebar (recruiter), dark mode,
    and route groups for recruiter vs public pages. Ready for Phase 2 (auth).
  </outcome>
</task>
